# Instructional Design

## Core Learning Philosophy

The site should not feel like content storage.

It should feel like guided self-education.

That means every course should be designed as a learning experience, not just as a sequence of articles. The structure should help the learner:

- understand what matters
- build momentum
- practice
- reflect
- retain
- continue

## Standard of Quality

The intended quality bar is comparable to or better than strong premium online learning platforms.

The standard is not:

- “good enough for a personal project”
- “interesting but shallow”
- “nicely written content with a quiz at the end”

The standard is:

- well-sequenced
- thoughtfully scaffolded
- grounded in strong sources
- useful in practice
- rewarding to move through

## Course Design Principles

### 1. Start Clear, Then Build Depth

Courses should begin with orientation and foundation before moving into complexity.

Learners should be able to answer:

- what this course is about
- why it matters
- how it is structured
- what kind of effort it asks for

Depth should increase gradually.

### 2. Lessons Should Not Feel Interchangeable

A common weakness in low-quality online learning is that every lesson feels the same length, same tone, same density, and same value.

This site should avoid that.

Lessons should vary appropriately based on:

- topic complexity
- conceptual density
- practice needs
- required examples
- learner friction points

### 3. Explanation Should Lead to Use

A lesson is not finished when the topic has been described.

It should also help the learner:

- apply
- inspect
- test
- compare
- reflect
- remember

This is why lessons include:

- objectives
- content
- checklist items
- reflection prompts
- knowledge checks
- notes

### 4. Reflection Should Be Meaningful

Reflection prompts should not sound like empty classroom homework.

They should feel:

- inviting
- personal
- imaginative where helpful
- useful for memory and transfer

Good reflection questions connect the topic to:

- the learner’s own workflow
- the learner’s own decisions
- the learner’s own ambitions

## Source Standards

Each course should be built around a source spine of roughly 3 to 5 high-value references.

Preferred source order:

1. official documentation
2. primary or canonical source texts
3. highly respected secondary practical references

Avoid weak source patterns:

- random blogs with low authority
- SEO content
- low-signal summaries
- trend-chasing references with weak substance

The resource library should reflect real trustworthiness, not just volume.

## Lesson Structure Expectations

Each lesson should usually include:

- a clear title
- a meaningful duration estimate
- well-defined objectives
- substantial instructional content
- checklist items tied to action or verification
- reflection questions
- a knowledge check when appropriate

For authored Markdown lessons, the current required sections are:

- `Objectives`
- `Lesson`
- `Checklist`
- `Reflection`
- `Quiz`

Optional:

- `Resources`

## Knowledge Check Standards

### 1. Assess Real Understanding

Questions should test:

- meaning
- comparison
- application
- design judgment
- practical understanding

Avoid:

- trivia
- obvious wording tricks
- purely superficial recall

### 2. Match Assessment Load to Topic Complexity

Complex lessons should have more assessment than simple lessons.

The current product standard aims for a more proportional relationship between topic depth and quiz depth.

### 3. Avoid Guessable Answer Patterns

Correct answers should not cluster in predictable positions.

The platform should deliberately avoid making “A or B” feel like a cheap shortcut.

### 4. Explanations Matter

A quiz is part of teaching, not just part of measuring.

Every answer explanation should help the learner:

- understand the concept more clearly
- notice the distinction the question was testing
- leave stronger than before they answered

## Writing Standards for Lessons

Lesson writing should be:

- clear
- intelligent
- accessible
- confident
- warm when appropriate
- not patronizing

Lessons should avoid:

- filler
- empty enthusiasm
- bloated throat-clearing
- jargon-heavy self-importance
- fake motivational language

Good lessons should feel like they respect the learner’s intelligence and time.

The authoring system should make it easier to produce this quality bar consistently, not lower it through convenience.

## Desired Learner Experience

The ideal learner reaction is:

- “This is more useful than I expected.”
- “This is actually teaching me something.”
- “I can feel the progression.”
- “This makes me want to keep going.”

## Future Instructional Opportunities

The site can eventually support more advanced instructional features while staying static:

- review lessons
- recap modules
- prerequisite guidance
- skill maps
- revision checkpoints
- practice-only views
- note-to-summary synthesis

Any such feature should strengthen learning, not simply add activity.
